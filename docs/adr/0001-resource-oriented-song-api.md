# Resource-oriented song API

The API will expose songs as the main resource using standard REST methods, with derived catalogue statistics available at `GET /api/songs/statistics`. Song collection reads will support query parameters such as `page`, `limit`, `genre`, and `q` so the list endpoint is bounded, filterable, and searchable from the start. This keeps CRUD operations conventional while making statistics clearly scoped to the song collection instead of introducing a vague top-level `stats` resource.

List reads return a pagination envelope containing `items`, `page`, `limit`, `totalItems`, and `totalPages`; create, read, and update operations return a single Song. Statistics are global to the Song Library rather than scoped to the current page, search, or genre filter. The persisted Song API contract is limited to catalogue metadata, including an optional `artworkUrl`; frontend-only presentation state such as duration, playback controls, volume, and selection state is not stored on Song records.
